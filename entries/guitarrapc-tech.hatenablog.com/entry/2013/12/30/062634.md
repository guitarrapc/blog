タイトル、公開日、更新日、作者、リンクアドレスを、公開日順に表示します。

<pre class="brush: powershell;">
Invoke-RestMethod -uri "https&#58;//gdata.youtube.com/feeds/api/videos?v=2&q=DSC+PowerShell" `
    | %{&#91;PSCustomObject&#93;@{
        Title = $_.title
        Published = $_.published
        Updated   = $_.Updated
        Author = $_.author.name
        Link = $_.content.src
    }} `
    | sort Published -Descending
</pre>

# 本題

さて検索はおまけで、本題はPowerShell でもヘルパーを用意したらどうだろうなのです、はい。

つまり、必要となる処理を事前に定義してしまう方向で考えようということです。

#### 対象のスクリプト

<pre class="brush: powershell;">
Invoke-RestMethod -uri "https&#58;//gdata.youtube.com/feeds/api/videos?v=2&q=DSC+PowerShell" `
    | %{&#91;PSCustomObject&#93;@{
        Title = $_.title
        Published = $_.published
        Updated   = $_.Updated
        Author = $_.author.name
        Link = $_.content.src
    }} `
    | sort Published -Descending
</pre>

この中の Foreach-Object で処理しているオブジェクトの合成。これを事前定義してしまうことができます。

#### Filter を利用する

パイプラインに入力されたストリームを順次処理する。これは、PowerShell において Filter で処理されるのが定石です。

Functionがストリーム経由でパイプラインに渡されたオブジェクト全体を1つとして処理していたにのに対して、Filter を使えばパイプラインへの入力をストリーム処理 -> 各オブジェクトを順次処理可能になります。簡単にいうと、process{} 節だけをもつFunctionなのですが、まぁ便利です。

で、今回のをFilterを使って、オブジェクト処理を定義するとこのようになります。

<pre class="brush: powershell;">
filter Filter-YoutubeApi
{
    &#91;PSCustomObject&#93;@{
        Title     = &#91;string&#93;$_.title
        Published = &#91;datetime&#93;$_.published
        Updated   = &#91;datetime&#93;$_.Updated
        Author    = &#91;string&#93;$_.author.name
        Link      = &#91;uri&#93;$_.content.src
    }
}
</pre>

これでどう変わるかというと、記述がこれで済みます。

<pre class="brush: powershell;">
Invoke-RestMethod -uri "https&#58;//gdata.youtube.com/feeds/api/videos?v=2&q=DSC+PowerShell" `
    | Filter-YoutubeApi `
    | sort Published -Descending `
</pre>

何度も登場するストリーム処理パターンを定義しておく。他言語でやる手法が、今ようやくPowerShellでも取り上げられつつあります。((今回は用途を限定しますが、汎用的な用途を今後示していければと考えています。))

# まとめ

近くいくつかのパターンをまとめる予定です。map や reduce などの処理をまとめておいて、Moduleに組み込む。こういったやり方でコレクションを処理しやすくなります。

ちなみに、Filter はPowerShell におけるパイプラインを通した繰り返し処理で最速の手段です。Foreach-Object や V3の省略記法 などよりずっと早いです。Where-Objectなどのコレクションフィルタリングでも示しています。
