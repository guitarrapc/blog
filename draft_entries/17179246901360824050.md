`<PackageReference Include>`は使うけど`<PackageReference Update>`は使わない、そう思っていた時期が私にもありました。今回は、`<PackageReference Update>`を使うと便利な例です。忘れそうなのでメモです。

[:contents]

## CPMとDirectory.Build.propsで全プロジェクトにパッケージを追加

Central Package Managementを使用しているプロジェクトで、Directory.Build.propsにパッケージを指定、Directory.Build.propsで全プロジェクトに指定したパッケージを追加できます。個別のプロジェクトに追加しなくていいので便利です。

```xml
<!-- Directory.Packages.props -->
<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
    <CentralPackageTransitivePinningEnabled>true</CentralPackageTransitivePinningEnabled>
  </PropertyGroup>
  <ItemGroup>
    <PackageVersion Include="Microsoft.SourceLink.GitHub" Version="8.0.0" />
  </ItemGroup>
</Project>
```

```xml
<!-- Directory.Build.props -->
<Project>
  <ItemGroup>
    <PackageReference Include="Microsoft.SourceLink.GitHub" PrivateAssets="all"/>
  </ItemGroup>
</Project>
```

例えばプロジェクトファイルは以下のようになっているとします。

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net10.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

</Project>
```

```cs
Console.WriteLine("Hello, World!");
```

この場合、Directory... で指定したMicrosoft.SourceLink.GitHubが利用されます。個別のcsprojにPackageReferenceを追加しなくても、Directory.Build.propsで全プロジェクトに指定したパッケージが利用できるのは便利です。

```shell
$ dotnet build
Restore complete (0.6s)
  Normal net10.0 succeeded (1.1s) → bin\Debug\net10.0\Normal.dll

Build succeeded in 2.6s
```

## どういうときに困るのか

ただ、時々、プロジェクトファイルでCPMを無効にしたくなる時があります。`<PackageReference Update>`はそういうときに使えます。

ライブラリ開発しているとプロジェクト参照ではなく、過去リリースしたNuGetパッケージで動作テストしたくなることがあります。この場合、プロジェクトファイルでCPMを無効にして、個別のプロジェクトファイルでPackageReferenceを追加する必要があります。例えば、`UseNuGetClient`をビルド時に指定するとCPMを無効にするようにして、NuGetパッケージを追加するようにしてみましょう。

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net10.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <PropertyGroup Condition="'$(UseNuGetClient)' != ''">
    <ManagePackageVersionsCentrally>false</ManagePackageVersionsCentrally>
  </PropertyGroup>
  <ItemGroup Condition="'$(UseNuGetClient)' != ''">
    <!-- Follow to the Directory.Packages.props version -->
    <PackageReference Include="MagicOnion.Client" Version="$(UseNuGetClient)"/>
  </ItemGroup>
</Project>
```

これでビルドすると、先にDirectory.Build.propsで指定した`Microsoft.SourceLink.GitHub`が追加されるものの、Directory.Build.propsではバージョンを指定していないため、ビルドエラーが発生します。困りました。

```shell
$ dotnet build -p:UseNuGetClient=6.1.4
    ./Override/Normal.csproj : error NU1015: The following PackageReference item(s) do not have a version specified: Microsoft.SourceLink.GitHub

Restore failed with 1 error(s) in 1.2s
```

この状況は、depsファイルを見ることで確認できます。Microsoft.SourceLink.GitHubのバージョンが指定されていないですね！

```shell
$ cat obj/プロジェクト名.csproj.nuget.dgspec.json
...省略

      "frameworks": {
        "net10.0": {
          "targetAlias": "net10.0",
          "dependencies": {
            "MagicOnion.Client": {
              "target": "Package",
              "version": "[6.1.4, )"
            },
            "Microsoft.SourceLink.GitHub": {
              "suppressParent": "All",
              "target": "Package",
              "version": "(, )"
            }
          },

...省略
```

では、NuGetパッケージを通常追加するときと同様に、csprojで`<PackageReference Include="Microsoft.SourceLink.GitHub" Version="8.0.0"/>` を追加すればいいかというと、これはDirectory.Build.propsと定義重複してエラーになります。

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net10.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <PropertyGroup Condition="'$(UseNuGetClient)' != ''">
    <ManagePackageVersionsCentrally>false</ManagePackageVersionsCentrally>
  </PropertyGroup>
  <ItemGroup Condition="'$(UseNuGetClient)' != ''">
    <PackageReference Include="MagicOnion.Client" Version="$(UseNuGetClient)"/>
    <!-- Follow to the Directory.Packages.props version -->
    <PackageReference Include="Microsoft.SourceLink.GitHub" Version="8.0.0"/>
  </ItemGroup>
</Project>
```

エラーからパッケージ定義が重複していることが分かります。

```shell
$ dotnet build -p:UseNuGetClient=6.1.4
./Bad/Normal.csproj : warning NU1504: Duplicate 'PackageReference' items found. Remove the duplicate items or use the Update functionality to ensure a consistent restore behavior. The duplicate 'PackageReference' items are: Microsoft.SourceLink.GitHub , Microsoft.SourceLink.GitHub 8.0.0.
    ./Bad\Normal.csproj : error NU1015: The following PackageReference item(s) do not have a version specified: Microsoft.SourceLink.GitHub

Restore failed with 1 error(s) and 1 warning(s) in 1.2s
```

## 対処

Directory.Build.propsで指定したPackageReferenceをプロジェクトファイルでバージョン指定して上書きしたいときは、`<PackageReference Include="...." />`ではなく`<PackageReference Update="..." Version="上書きしたいバージョン" />`を使いましょう。

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net10.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <PropertyGroup Condition="'$(UseNuGetClient)' != ''">
    <ManagePackageVersionsCentrally>false</ManagePackageVersionsCentrally>
  </PropertyGroup>
  <ItemGroup Condition="'$(UseNuGetClient)' != ''">
    <!-- Follow to the Directory.Packages.props version -->
    <PackageReference Include="MagicOnion.Client" Version="$(UseNuGetClient)"/>
    <PackageReference Update="Microsoft.SourceLink.GitHub" Version="8.0.0"/>
  </ItemGroup>

</Project>
```

ビルドが成功していますね。

```shell
dotnet build -p:UseNuGetClient=6.1.4
Restore succeeded with 1 warning(s) in 2.4s
    ./Good/Normal.csproj : warning NU1902: Package 'MessagePack' 2.5.129 has a known moderate severity vulnerability, https://github.com/advisories/GHSA-4qm4-8hg2-g2xm
  Normal net10.0 succeeded with 1 warning(s) (4.2s) → bin\Debug\net10.0\Normal.dll
    ./Good/Normal.csproj : warning NU1902: Package 'MessagePack' 2.5.129 has a known moderate severity vulnerability, https://github.com/advisories/GHSA-4qm4-8hg2-g2xm

Build succeeded with 2 warning(s) in 7.5s
```

## まとめ

もし、どこかですでに`<PackageReference Include="..."/>`と定義したパッケージをプロジェクトファイルで上書きしたいときは、`<PackageReference Update="..." Version="X.Y.Z"/>`を使うと上書きできます。状況的に、Directory.Build.propsやカスタムprops読み込みを使っていない限りは遭遇しないです。ただ、csproj管理の効率化のためにDirectory.Build.propsを使っているときは、思いがけず遭遇する可能性があります。

今ならこういうのはLLM聞けば? ってなるのですが、割とこういうファイルを探索する系はLLMもすんなり答えにたどり着かないことがあります。基礎知識ということで。
